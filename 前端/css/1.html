    # 盒子模型

    # 盒子模型的认识

    1. 基本概念：标准盒模型和IE模型（怪异盒模型）
    2. 标准盒模型和怪异盒模型的区别
    3. css设置宽高
    4. js设置宽高
    5. 用盒模型解释边距重叠
    6. BFC（BFC的基概念，触发BFC的场景，还有IFC），解决边距重叠

    # 一、标准盒模型和IE模型

    content、paddin、border、margin

    # 二、二者的区别

    内容的计算方式不一样 不同点：宽度计算方式不一样

    • 标准盒模型 width = content
    • IE模型 with = cnotent + padding + border 注：设置的宽度是盒子的宽度，只是content的宽度不一样 相同点：高度计算方式一样

    # 三、css设置宽高

    css3新增属性 box-sizing 可以设置盒子的类型
box-sizing: content-box; 标准盒模型
box-sizing: border-box; IE模型

    # 四、js设置宽高

    1. dom.stye.width/height 只能取到行内样式的宽和高，style标签中和link外链的样式取不到
    2. dom.currentStyle.width/height 取到的是最终渲染的宽和高，只有IE浏览器支持此属性
    3. windom.getComputed(dom).width/height 同2 但是多浏览器支持，IE9以上支持
    4. dom.getBundingClientRect.width/height 也是得到渲染后的宽和高，大多数浏览器支持。IE9以上支持，除此外还可以取到相对于视窗的上下左右的距离

    # 五、用盒模型解释边距重叠

    当两个垂直外边距相遇时，他们将形成一个外边距，合成的外边距等于两个发生外边距重叠的外边距中的外边距较大者 注：只有普通文档流中的的块框才会发生外边距重叠。行内框、浮动框或绝对定位之间的的外边距不会重叠

    # 六、BFC

    BFC（Block Formatting Context）：块级格式上下文 BFC决定了元素如何对其内容进行定位，以及与其他元素的关系和相互作用。 BFC提供了一个环境，HTML元素在这个环境中按照一定的规则进行布局。 一个环境中的元素不会影响到其他环境中的布局。

    BFC的原理（渲染规则）

    1. BFC垂直方向的元素会发生边距重叠。 属于不同的BFC外边距不会发生重叠
    2. BFC的区域不会与浮动元素的布局重叠。
    3. BFC的容器一个独立的容器，外面的元素和里面的元素不会相互影响。
    4. 计算BFC高度的时候，浮动元素也会参与计算（清除浮动）。

    触发BFC：

    1. overflow 不为 visible
    2. float值 不为 none
    3. postion 不为 static或relative
    4. display 属性为 inline-blocks，table，table-cell，table-caption，flex，inline-flex
