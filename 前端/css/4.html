    # BFC

    # 一、常见定位方案

    在聊BFC之前，我们可以先来了解一下常见的定位方案，定位方案是控制元素的布局。

    1. 普通流（normal flow）（文档流、流式布局）

    在普通流中，元素按照其HTML的先后位置在上而下布局，在这个过程中，行内元素水平排列，直到当行被占满后换行，块级元素则会被渲染为完整的一个新行，除非另外指定，否则所有元素默认都是普通流定位。

    1. 浮动（float）

    在浮动布局中，元素首先按照文档流中位置进行布局，然后根据浮动的方向，尽可能的向左边或者右边偏移，其效果与文字环绕相似。

    1. 绝对定位（absolute position）

    在绝对定位布局中，元素会脱离文档流，所以绝对定位不会对兄弟元素造成影响，而元素的具体位置由绝对定位的坐标决定。

    # 二、BFC

    BFC（Block Formatting Context，块级格式上下文），属于普通流。 BFC可以理解一定条件下触发的一种case，是一个独立的容器，容器里面的元素不会在布局上影响外面的元素，外面的元素也不回影响里面的元素。

    # 三、触发BFC

    • body 根元素
    • overflow 非 visible
    • 浮动元素：float 非 none
    • 绝对定位元素：position：absolute、fixed
    • display 为 inline-block、table、table-cell、table-caption、flex、inline-flex

    # 四、BFC特性及应用

    1. 同一个BFC下外边距会发生折叠
    2. BFC可以包含浮动的元素（清除浮动）

    # 五、清除浮动的方法

    1. 额外标签法：（在最后一个浮动标签后，新加一个标签，给其设置clear：both；）不推荐
    2. 父级添加overflow属性（父元素添加overflow:hidden）（不推荐）
    3. 使用after伪元素清除浮动（推荐使用） .clearfix:after{/伪元素是行内元素 正常浏览器清除浮动方法/ content: ""; display: block; height: 0; clear:both; visibility: hidden; }
    4. 使用before和after双伪元素清除浮动 .clearfix:after,.clearfix:before{ content: ""; display: table; } .clearfix:after{ clear: both; }
