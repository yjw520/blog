    # plugin介绍

    # 一、认识plugin

    1. webpack的另一个核心是plugin
    • loader是用于特定的模块类型进行转换
    • plugin可以用于执行更加广泛的任务，比如打包优化、资源管理、环境变量注入等 ;

    # 二、常见plugin

    1. CleanWebpackPlugin
    • 之前打包过后每次修改了一些配置，重新打包时，都需要手动删除dist文件夹
      • 我们可以借助于一个插件来帮助我们完成，这个插件就是CleanWebpackPlugin ;
    1. HtmlWebpackPlugin
    • 我们进行项目部署时，必然需要入口文件
    • 对HTML进行打包处理我们可以进行使用另外一个插件：HtmlWebpackPlugin ;
    • 这个文件是如何生成的呢？
      • 默认情况下是根据ejs的一个模版来生成的
      • 在html-webpack-plugin的源码中，，有一个defaul_index.ejs模块
    • 如果我们想在自己的模块中加入一些比较特殊的内容
      • 比如添加一个noscript标签，在用户的JavaScript被关闭时，给予响应的提示
      • 比如在开发vue或者react项目时，我们需要可以一个可以挂载的根标签<div id="app"></div> ;
      • 上面的代码中，会有一些类似这样的语法<% 变量 %>，这个是EJS模块填充数据的方式
      • 在配置HtmlWebpackPlugin时，我们可以添加如下配置：
        • template：指定我们要使用的模块所在的路径
        • title：在进行htmlWebpackPlugin.options.title读取时，就会读取到该信息 ;

    1. DefinePlugin（webapack内置插件） +;

    2. CopyWebpackPlugin

    • 在vue的大包过程中，如果我们将一些文件放到public目录下，那么这个目录会被复制到dist文件夹中
      • 这个复制的功能，我们可以使用CopyWebpackPlugin来完成
    • 安装CopyWebpackPlugin插件
      npm install copy-webpack-plugin -D
      1
    • 接下来配置CopyWebpackPlugin即可
      • 复制的规则在patterns中设置
      • from：设置从哪一个源中开始复制
      • to：复制的位置，可以省略，会默认复制到打包的目录下
      • globOptions：设置一些额外的选项，其中可以编写需要忽略的文件
        • .DS_Store：mac目录下自动生成的一个文件
        • index.html：也不需要复制，因为我们已经通过HtmlwebpackPlugin完成了index.html的生成
      new CopyWebpackPlugin({
  patterns: [
    {
      from: "public",
      globOptions: {
        ignore: [
          '**./.DS_Store',
          '**/index.html'
        ]
      }
    }
  ]
});
